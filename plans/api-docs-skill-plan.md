# Plan: `api-docs` Skill — Per-Endpoint API Documentation Generator

## Context

ShopStack currently has no API documentation: `docs/` is empty, no Swagger / OpenAPI is wired up, and `README.md` is one line. The Django app `stackapp` exposes ~10 endpoints under `/api/` whose request/response shapes are only discoverable by reading `views.py`, `serializers.py`, and `urls.py` together. Developers (and reviewers) repeatedly re-derive this when working on a single endpoint.

This skill, `api-docs`, accepts either a URL path (e.g. `/api/orders/`) or a view class name (e.g. `OrderListCreateView`), statically analyzes the relevant view + serializer + model files, and emits a professional ShopStack-branded `.docx` reference document at `docs/api/<endpoint-slug>.docx` — one document per API. No live requests. No OpenAPI. No shared aggregated file.

Layout mirrors the existing `.claude/skills/pr-review/SKILL.md` so contributors find a familiar structure.

## Rendering approach: python-docx (not pandoc)

The skill renders Word documents directly via the **python-docx** library (pure Python, pip-installable into `~/genv`). Earlier iterations considered pandoc + a binary `reference-template.docx`, but pandoc requires system-level install (`sudo apt`) that isn't available in every developer environment. python-docx works anywhere a virtualenv works, and keeps all branding in version-controlled code rather than a binary asset.

## Files

1. `.claude/skills/api-docs/SKILL.md` — skill definition (frontmatter + step-by-step instructions).
2. `.claude/skills/api-docs/generate_docx.py` — reusable renderer. Reads a JSON spec on stdin, writes a styled `.docx` to the path passed as argv[1]. Owns all styling: ShopStack blue (#1F4E79), Calibri body, Consolas code, banded blue-header tables, A4 + 2.5cm margins, page header with logo + "ShopStack API Reference", footer "Confidential — Generated by api-docs skill".
3. `.claude/skills/api-docs/shopstack-logo.png` — branding asset (~20KB PNG). Used on the cover page and in the page header.
4. `docs/api/.gitkeep` — keeps the output directory in the repo.

## Workflow (per invocation)

1. **Resolve endpoint.** Parse `shopstack/stackapp/urls.py` and `auth_urls.py`. Match the user's argument against `full_path` (URL starts with `/`) or `view_class` (otherwise). Ambiguous → `AskUserQuestion`. No match → list candidates and abort.
2. **Locate artifacts.** `Grep` for `class <ViewClass>`. Read view class body (base class, `get_queryset`, `get_permissions`, filter backends, custom verb overrides, `http_method_names`). Resolve serializer → read `Meta.model` / `Meta.fields` / validators / overrides. Resolve model → read field types, FKs, choices.
3. **Derive facts.** Map base class to HTTP methods. Per method derive auth, path params, query params (incl. `query_params.get` reads), request body (writable fields), response body (all fields, recurse one level), errors (scan view for `Response(..., status=...)`, `raise ValidationError`, `Http404`). Synthesize JSON examples by field type. Wrap list responses in DRF `PageNumberPagination` envelope.
4. **Build JSON spec.** Construct a single JSON object (schema documented in `generate_docx.py` module docstring). Write to `/tmp/api-docs-<slug>.json` via the `Write` tool — mirrors how `pr-review` writes `/tmp/pr-review-payload.json`.
5. **Render.** Activate `~/genv`, verify `python-docx` is importable, run `python3 .claude/skills/api-docs/generate_docx.py docs/api/<slug>.docx < /tmp/api-docs-<slug>.json`.
6. **Recap.** Print endpoint, view/serializer/model paths, method count, field counts, output path.

## Rebranding for other organizations

Two edits, no other code changes:

1. Replace `.claude/skills/api-docs/shopstack-logo.png` with the new org's logo.
2. In `generate_docx.py`, change `BRAND` / `BRAND_HEX` constants and the `brand_text` / `footer_text` strings inside `_header_footer()`.

`SKILL.md` and the skill's invocation surface stay unchanged.

## Guardrails

- Static analysis only — never start the dev server, never hit a URL.
- Read-only on `views.py`, `serializers.py`, `models.py`, `urls.py`.
- One endpoint per invocation.
- Abort with a clear hint if `python-docx`, `generate_docx.py`, or `shopstack-logo.png` is missing.
- Abort if the resolved view is not a DRF generic or `APIView` subclass.
- Overwrite the `.docx` on re-run; no suffixes.
